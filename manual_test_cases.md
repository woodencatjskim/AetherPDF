# AetherPDF 수동 검증 시나리오 및 가이드 (Manual Test Cases)

본 문서는 오프라인 단독 실행형 초경량 PDF 편집기 **AetherPDF**의 주요 기능과 사용 편의성(반응성, UI 동기화, 다이얼로그 연동)을 수동으로 검증하기 위한 상세 가이드라인입니다.

---

## 📋 1. 테스트 전 준비 사항
1. **바이너리 위치**: `c:\Users\jskim\project\AetherPDF\dist\AetherPDF.exe`
2. **테스트 대상 OS**: Windows 10 / 11
3. **준비물**: 텍스트가 포함된 임의의 PDF 파일 1개, 병합 테스트용 외부 PDF 파일 1개
4. **오프라인 환경 검증**: PC의 네트워크(Wi-Fi, 이더넷)를 차단한 후 실행하여 완벽한 오프라인 단독 구동이 보장되는지 확인합니다.

---

## 🧪 2. 기능별 상세 테스트 케이스

### 🔍 TC-01: PDF 로드 및 기본 뷰어 탐색
*   **목적**: PDF 파일을 정상적으로 불러오고 UI 요소들이 지연 없이 부드럽게 반응하는지 검증합니다.
*   **테스트 절차**:
    1. `dist/AetherPDF.exe`를 더블클릭하여 실행합니다.
    2. 상단 툴바의 **파일 열기 아이콘**을 클릭하거나 단축키 `Ctrl + O`를 입력합니다.
    3. Windows 표준 파일 선택 창이 표시되면 준비한 PDF 파일을 선택하여 엽니다.
    4. 마우스 휠 및 스크롤바를 이용해 페이지를 하단으로 스크롤해봅니다.
    5. 좌측 사이드바 패널에 썸네일 이미지가 순서대로 생성되는지 확인합니다.
*   **합격 기준**:
    *   [ ] PDF 파일이 즉시 렌더링되며 윈도우 창 크기에 맞게 깔끔하게 배치됨.
    *   [ ] 스크롤 시 화면 찢어짐이나 큰 렌더링 버벅임 없이 부드럽게 페이지가 넘어감.
    *   [ ] 좌측 사이드바에 각 페이지의 미니 썸네일이 정상적으로 노출됨.

---

### ✍️ TC-02: 본문 텍스트 수정 [B]
*   **목적**: PDF 내부의 텍스트 레이어를 안전하게 Redaction 처리하여 지우고 새로운 내용으로 자연스럽게 교체하는지 검증합니다.
*   **테스트 절차**:
    1. 상단 툴바에서 **텍스트 편집 도구**를 활성화합니다.
    2. 본문 영역의 문장이나 단어 위에 마우스 커서를 올렸을 때 편집 가능 블록 경계선(하이라이트 점선)이 표시되는지 확인합니다.
    3. 편집하고 싶은 텍스트 블록을 **더블클릭**합니다.
    4. 편집 다이얼로그가 표시되면 기존 텍스트를 지우고 한글 및 영문 텍스트를 입력합니다.
    5. **적용** 버튼을 누릅니다.
*   **합격 기준**:
    *   [ ] 기존에 있던 글자가 잔상 없이 투명하고 깔끔하게 지워짐.
    *   [ ] 새로 입력한 텍스트가 원본 텍스트의 폰트 크기, 스타일(Bold 등), RGB 색상을 최대한 보존하며 동일한 시작 위치에 오버레이됨.
    *   [ ] 텍스트 교체 후 다른 페이지로 이동했다가 돌아와도 렌더링이 깨지지 않음.

---

### 🗂️ TC-03: 페이지 관리 및 동적 편집 [C]
*   **목적**: 페이지 순서 변경, 회전, 삽입 및 삭제 작업이 유기적으로 이루어지며 화면과 실시간 동기화되는지 검증합니다.
*   **테스트 절차**:
    *   **순서 변경 (Drag-and-Drop)**:
        1. 좌측 썸네일 목록에서 임의의 페이지 썸네일을 마우스 드래그하여 다른 페이지 사이로 이동시킵니다.
    *   **페이지 회전 / 삭제**:
        1. 특정 페이지 썸네일을 우클릭하여 콘텍스트 메뉴를 호출합니다.
        2. **시계방향 90도 회전**을 선택합니다.
        3. 다시 우클릭하여 **페이지 삭제**를 실행합니다.
    *   **빈 페이지 삽입 / 외부 PDF 병합**:
        1. 썸네일 우클릭 콘텍스트 메뉴에서 **빈 페이지 삽입**을 수행합니다.
        2. **외부 PDF 병합(Merge)**을 클릭하여 다른 PDF를 지정한 위치에 덧붙여 봅니다.
*   **합격 기준**:
    *   [ ] 썸네일을 드래그 앤 드롭했을 때, 마우스 릴리즈 즉시 메인 뷰어의 본문 페이지 순서와 썸네일 목록이 정밀하게 동기화됨.
    *   [ ] 회전 시 메인 뷰어와 썸네일 뷰어 모두 즉시 90도 회전된 형상으로 리렌더링됨.
    *   [ ] 페이지 삽입, 삭제 및 병합된 페이지들이 번호 손실 없이 실시간으로 재배치됨.

### 🎨 TC-04: 네온 주석 및 Ink 드로잉 [A]
*   **목적**: 주석 속성 패널을 통해 색상, 두께, 투명도를 조절하고, 텍스트 주석 및 펜 드로잉이 직관적으로 렌더링되는지 확인합니다.
*   **테스트 절차**:
    *   **텍스트 주석 (형광펜, 밑줄, 취소선)**:
        1. 상단 툴바의 **주석 도구**를 선택합니다. 우측에 **주석 속성 패널**이 자연스럽게 나타나는지 확인합니다.
        2. 속성 패널에서 프리셋 컬러(Neon Cyan, Neon Yellow 등)를 선택하고 투명도를 50%로 설정합니다.
        3. 메인 뷰어 본문 글자 위를 마우스로 드래그합니다.
    *   **Ink (자유 곡선 펜) 드로잉**:
        1. 속성 패널에서 **펜(Pen) 도구**를 선택합니다.
        2. 선 두께 슬라이더를 `5px`로, 투명도를 `80%`로 설정합니다.
        3. 본문 여백에 마우스 왼쪽 버튼을 누른 채 원형이나 자유 선을 부드럽게 그려봅니다.
*   **합격 기준**:
    *   [ ] 텍스트 주석 드래그 시 Neon Cyan 점선 사각형 피드백 영역이 실시간으로 노출되며, 드래그를 놓으면 그 영역에 걸친 실제 텍스트 글자 범위를 정교하게 인식해 오버레이가 생성됨.
    *   [ ] 마우스로 펜을 그릴 때 끊김이나 드로잉 인풋 지연(Lag) 없이 매끄러운 궤적 피드백이 렌더링됨.
    *   [ ] 속성 패널의 컬러 프리셋 외에도 Custom Color 버튼 클릭 시 Windows 표준 `QColorDialog` 창이 뜨고 색상이 연동됨.

### 💾 TC-05: 점진적 및 압축 최적화 저장
*   **목적**: 저장 후 데이터가 소실되지 않고 다른 PDF 리더에서도 편집 상태가 영구적으로 보존되며 파일 용량이 최적화되는지 검증합니다.
*   **테스트 절차**:
    1. 주석과 텍스트를 여러 개 수정한 상태에서 **파일 저장 버튼**을 누르거나 `Ctrl + S`를 눌러 원본 파일에 덮어씁니다.
    2. 상단 툴바의 **다른 이름으로 저장**을 누르거나 `Ctrl + Shift + S`를 누릅니다.
    3. 다른 이름으로 저장 대시보드 다이얼로그에서 **"압축 최적화 저장 (Deflate & Garbage Collect)"** 체크박스를 활성화하고 새 이름으로 내보냅니다.
    4. 저장된 두 PDF 파일을 닫은 뒤, **Adobe Acrobat Reader** 또는 **Microsoft Edge 브라우저**로 열어봅니다.
*   **합격 기준**:
    *   [ ] 다른 PDF 리더에서 열었을 때, AetherPDF에서 수정한 본문 글자와 추가한 형광펜 주석, 자유 잉크 낙서 등이 한치의 오차도 없이 동일한 위치에 깔끔하게 표기됨.
    *   [ ] 압축 최적화 저장된 파일의 용량이 무손실 압축으로 인해 일반 저장본 대비 불필요한 메타데이터가 완벽 제거되어 가볍게 생성됨.

## 📈 3. 성능 및 미적 가이드라인 자가 진단
*   **초경량 오프라인 구동**: 실행 클릭 후 UI 초기 기동 속도가 1.5초 이내여야 함.
*   **에테르 다크 테마(Aether Dark Theme)**: 네온 시안 및 네온 마젠타 포인트 컬러가 다크 그레이 계층 배경과 Harmonious하게 조화를 이루며 세련된 프리미엄 룩앤필을 구현하는지 확인합니다.
